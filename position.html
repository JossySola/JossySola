POSITION
Concept Description Graphic
The DEFAULT LAYOUT makes ELEMENTS OCCUPY A ROW of the document from left to right, start at the top and move down the page.

STATIC

Technicality:
This is the default position applied to all elements in HTML.

How I see it:
The element is grounded and CANNOT MOVE in its corresponding row space.

ROW SPACE

The properties: top, bottom, left, right do NOT work with this position.

RELATIVE

Technicality:
The element is positioned RELATIVE to its normal position.

How I see it:
The element can walk around its row space (assigned row space of the document).

ROW SPACE

The properties: top, bottom, left and right are used to move the element around within its assigned row space in the document.

ABSOLUTE

Technicality:
The element is positioned relative to the nearest positioned ancestor.

How I see it:
The absolute element doesn't own a row in the document because he cannot afford it.
Lil Absolute still lives within his dad row space (parent element containing him) and moves around his dads.

ROW SPACE

Parent
Sibling

Absolute elements can OVERLAP elements using the properties: top, bottom, left and right.

FIXED

Technicality:
The element is positioned relative to the VIEWPORT, which means it always stays in the same place even if the page is scrolled.

How I see it:
The element is GLUED directly in the document and does NOT own any row within the document.
Let's pretend this is a website, I'm the fixed one...

STICKY

Technicality:
The element is positioned based on the user's scroll position. A sticky element toggles between relative and fixed.

How I see it:
Sticky likes to hang out its row.
He owns a row in the neighbourhood and is able to move around inside it BUT when the scroll reaches him, he glues himself to the document to visit other rows.
